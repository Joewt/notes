

```
__construct() -- 构造函数 在创建新对象的时候调用
__destruct() -- 析构函数 在某个对象的所有引用被删除的时候或者对象被显式销毁时调用
__set() -- 在给不可访问属性赋值时，会被调用
__get() -- 读取不可访问属性的值时，会被调用
__call() -- 在对象中调用一个不可访问方法时
__callStatic() -- 在静态上下文中调用一个不可访问方法时
__isset() -- 当对不可访问属性调用 isset() 或 empty() 时
__unset() --  当对不可访问属性调用 unset() 时
__sleep() -- serialize()函数调前会先调用__sleep() 
__wakeup() -- 与之相反，unserialize() 会检查是否存在一个 __wakeup() 方法。如果存在，则会先调用 __wakeup 方法，预先准备对象需要的资源。
__toString() -- 当尝试打印类的时候回调用 比如 echo $obj 
__invoke() -- 当尝试以调用函数的方式调用一个对象时 会被调用
__set_state() -- 起当调用 var_export() 导出类时,会被调用
__clone() -- 当复制完成时，如果定义了 __clone() 方法，则新创建的对象（复制生成的对象）中的 __clone() 方法会被调用，可用于修改属性的值（如果有必要的话）。
__debugInfo() -- 当使用var_dump()一个对象的时候 会被调用



```
